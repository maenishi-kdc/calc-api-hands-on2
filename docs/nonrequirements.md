# 非機能要件

## 環境・運用
- リージョン: Azure 東日本 (Japan East)
- プラン: Azure Functions - Consumption Plan
- ランタイム: Python 3.11
- リソースグループ: calc-api-hands-on-test-maenishi
- デプロイ: 手動デプロイ（CI/CD は導入しない）

## 可用性・性能
- 最小構成で運用するため可用性目標や SLA は設定しない
- レイテンシ要件は設定しない（デフォルトの挙動で可）
- Consumption Plan に起因するコールドスタートを許容する

## セキュリティ
- 認証は不要（パブリック公開）
- パブリック公開に伴うリスクは受容するが、必要であれば後からアクセス制限を追加可能
- 入力検証を行い不正なパラメータでのサーバーエラーを防止する

## ログ・監視・保守
- 最小限のログ（エラー発生時に例外とメッセージをログ出力）
- 監視・アラートは設定しない（今後の拡張で追加可能）
- ログ保持期間の要件は無し

## コスト
- Consumption Plan を利用し、使用量に応じた課金モデルを前提とする
- 常時稼働のインスタンスを想定しないことでコストを抑制する

## スケーラビリティ
- Consumption Plan の自動スケールに依存する。短時間の高負荷時はインスタンスが増える可能性がある
- 長時間の高負荷・高スループットは未検証

## テスト要件
- 単体テスト（ユニットテスト）を実装する（`tests` ディレクトリに配置）
  - 正常系: 乗算・除算の結果確認
  - 異常系: 非整数入力、未指定、0 除算
- E2E テストは必須ではないが、手動でブラウザ確認を行う

## 制約事項・備考
- Python 3.11 を使用するが、Azure Functions のランタイムサポートをデプロイ前に再確認することを推奨
- 認証・監視・詳細な SLA が不要なため、最小実装に集中する
- 今後の拡張で JSON レスポンスや認証、CI/CD を追加する余地を残す
